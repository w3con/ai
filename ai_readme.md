# Командный слой — справочник

Четыре слэш-команды, которыми Alex управляет режимом моей работы. Каждая команда — это файл-скилл `skills/<имя>/SKILL.md`, и имя команды равно имени папки. Текст, который Alex пишет после команды, приходит в скилл как подстановка `$ARGUMENTS`. Команды — управляющие сигналы Alex; сам я их не запускаю (у всех в шапке `disable-model-invocation: true`).

| Команда | Файл | Что значит |
|---|---|---|
| `/q` | `skills/q/SKILL.md` | Дальше вопрос — отвечать, а не действовать. |
| `/p` | `skills/p/SKILL.md` | Выдать фазированный план и записать в `ai/plans/<слаг>.md`. |
| `/s` | `skills/s/SKILL.md` | Отдать текущий план на проверку фазировки агенту `scope`. |
| `/f` | `skills/f/SKILL.md` | Дописать запись о промахе в `harness/fuckups.md`. |

Команда привязана ровно к одному поведению, а текст после неё — только аннотация к ней. Это эргономика и контроль поверх build-дисциплины (хук-гейт `plan-gate` плюс проверяющий агент `scope`), а не вместо неё. Команды KB-цикла `/n` и `/l` сняты вместе с loop-аппаратом (см. `decisions.md`, 2026-06-24): мышление и база знаний ведутся в отдельном инструменте, а Claude Code — сборщик.
