# Командный слой — справочник

Шесть слэш-команд, которыми Alex управляет режимом моей работы. Каждая команда — это файл-скилл `skills/<имя>/SKILL.md`, и имя команды равно имени папки. Текст, который Alex пишет после команды, приходит в скилл как подстановка `$ARGUMENTS`. Команды — управляющие сигналы Alex; сам я их не запускаю (у всех в шапке `disable-model-invocation: true`).

| Команда | Файл | Что значит |
|---|---|---|
| `/n` | `skills/n/SKILL.md` | Войти в loop по предмету: открыть рабочую сессию (`current.md`, `Status: active`) и подгрузить его уже существующие KB, решения и план. KB не создаёт — это `new-kb`. |
| `/q` | `skills/q/SKILL.md` | Дальше вопрос — отвечать, а не действовать. |
| `/l` | `skills/l/SKILL.md` | Припарковать побочную мысль как parked-сессию (`current.md`, `Status: parked`), не выходя из текущей. KB не создаёт. |
| `/p` | `skills/p/SKILL.md` | Выдать фазированный план и записать в `ai/plans/<слаг>.md`. |
| `/s` | `skills/s/SKILL.md` | Отдать текущий план на проверку фазировки агенту `scope`. |
| `/f` | `skills/f/SKILL.md` | Дописать запись о промахе в `harness/fuckups.md`. |

Команда привязана ровно к одному поведению, а текст после неё — только аннотация к ней. Смысл слоя — убрать моё толкование из цикла: команда значит одно, и гадать «он хочет действия или просто обсуждает» больше не нужно. Это эргономика и контроль поверх механизма (хук-гейт плюс проверяющий агент `scope`), а не вместо него.
