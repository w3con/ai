The Validité knowledge base (`~/Documents/Validite/kb/`, 324 docs across 10 collections) has
a generated index `_index.md` — one line per document with a summary, regenerated by
`ai/bin/kb-index`. Alex expects lookups to START there (2026-07-08: «у тебя же есть индекс
KB, зачем ты grep делаешь?»).

**Why:** the index summaries answer "does a document about X exist at all" in one read;
blind grep produces noisy hits (entity lists, tags) and misses documents whose wording
differs from the search term.

**How to apply:** Read `_index.md` (or grep the INDEX for the topic word) → open only the
matched documents. Fall back to full-text grep only when the index has no plausible match
and the fact might live inside a doc body.
