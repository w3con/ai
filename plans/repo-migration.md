## Task statement

Сделать `/Volumes/SSD/Dev/ai` единственным источником истины для всего AI-глобального
(харнес, user-память, агенты, хук-гейт, кросс-проектный тулинг) и завести его в `~/.claude/`
симлинками так, чтобы Claude Code в любой проектной папке подтягивал правила, память и агентов
именно из репо, а не из разрозненных мест. Сейчас всё разбросано: харнес лежит ошибочно в
проектной папке `Validité/ai/harness/`, user-память в `~/.claude/memory/`, агент `scope` в
`~/.claude/agents/`, хук в `~/.claude/hooks/`, глобальный `CLAUDE.md` в `~/.claude/`. После
переезда оригиналы становятся симлинками внутрь репо, репо версионируется через git и едет
между машинами, а проектные папки хранят только проектное.

Почему симлинки, а не копии: Claude Code читает правила/память/агентов из фиксированных путей
внутри `~/.claude/` (`CLAUDE.md`, `memory/`, `agents/`). Мы не можем переучить его читать из
`/Volumes/SSD/...`, поэтому оставляем эти пути на месте, но делаем их симлинками в репо. Репо —
истина; `~/.claude/<X>` — указатель на неё.

## Критический механизм (как Claude подхватит репо)

Claude Code авто-загружает `~/.claude/CLAUDE.md`, `~/.claude/memory/` (через `@import` в
CLAUDE.md) и `~/.claude/agents/`. Эти три пути после переезда становятся симлинками в репо —
этого достаточно, чтобы правила/память/агенты тянулись из репо в любом проекте. `harness/` не
авто-загружается — это справочные документы, на которые ссылаются правила; ему достаточно
просто лежать в репо. `@import`-строки в `CLAUDE.md` используют абсолютные пути
`/Users/tknff/.claude/memory/...` — после симлинка `~/.claude/memory → repo/memory` они
по-прежнему резолвятся, переписывать импорты не нужно.
